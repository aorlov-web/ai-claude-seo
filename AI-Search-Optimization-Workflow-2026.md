# 1. Що таке AI Search Optimization

AI Search Optimization — це підхід до оптимізації сайту, при якому контент, структура, технічна база, довіра, експертність і семантика готуються не лише для класичного Google Search, а й для AI-систем, які формують готові відповіді користувачам.

У 2026 році задача сайту — не просто бути в індексі Google, а стати джерелом, яке AI може:

- зрозуміти;
- процитувати;
- використати у відповіді;
- рекомендувати користувачу;
- пов’язати з конкретною темою, послугою, брендом або локацією.

---

# 2. Головна ціль оптимізації

Ціль AI SEO / GEO:

> Зробити сайт таким, щоб Google AI Search і AI Overviews могли легко витягнути з нього відповідь, довіряти цій відповіді та показати сайт як корисне джерело.

Для цього сайт має бути:

- технічно стабільним;
- швидким;
- зрозумілим;
- структурованим;
- експертним;
- entity-based;
- локально релевантним;
- комерційно зрозумілим;
- з хорошим UX;
- без технічного хаосу.

---

# 3. Як AI Search працює на практиці

AI-пошук не просто показує 10 синіх посилань. Він аналізує багато джерел, витягує з них фрагменти, порівнює дані, шукає узгоджені факти та формує відповідь.

AI оцінює:

- чи зрозуміло, хто стоїть за сайтом;
- чи є на сайті чітка структура;
- чи є відповіді на конкретні питання;
- чи є експертність;
- чи є довіра;
- чи є зв’язки між темами;
- чи є ціни, приклади, кейси, FAQ;
- чи швидко сайт завантажується;
- чи не прихований контент через JavaScript;
- чи не виглядає сайт як масовий AI-спам.

# 5. Технічний аудит сайту

## 5.1 Core Web Vitals

Перевірити:

- LCP;
- INP;
- CLS;
- TTFB;
- FCP;
- Speed Index.

Цільові показники:

| Метрика | Бажане значення | Пріоритет |
|---|---|---|
| LCP | до 2.5 сек | Core Web Vital |
| INP | до 200 мс | Core Web Vital |
| CLS | до 0.1 | Core Web Vital |
| TTFB | до 800 мс | Технічний |
| FCP | до 1.8 сек | Додатковий |
| Speed Index | до 3.4 сек | Додатковий |

> **Примітка:** LCP, INP, CLS — офіційні Core Web Vitals Google (поле CrUX). FCP і Speed Index — lab-метрики з Lighthouse, не враховуються в рейтингу напряму, але впливають на сприйняття швидкості.

## 5.2 Що виправити

- оптимізувати зображення;
- перейти на WebP або AVIF;
- увімкнути lazy loading;
- прибрати зайвий JavaScript;
- мінімізувати CSS;
- налаштувати кешування;
- увімкнути Brotli або gzip;
- перевірити хостинг;
- оптимізувати шрифти;
- видалити зайві плагіни;
- прибрати блокуючі ресурси;
- перевірити мобільну швидкість.

## llms.txt — новий стандарт для AI-систем

`llms.txt` — це файл у корені сайту (аналог `robots.txt`, але для великих мовних моделей), який підказує AI-системам що і як читати на сайті.

Розміщується за адресою: `https://ваш-сайт.ua/llms.txt`

Приклад мінімального `llms.txt`:
```txt
# llms.txt
> Назва компанії: Ваш бізнес
> Опис: Коротко про що сайт і послуги

## Основні сторінки
- [Головна](https://ваш-сайт.ua/)
- [Послуги](https://ваш-сайт.ua/services/)
- [Про нас](https://ваш-сайт.ua/about/)
- [Контакти](https://ваш-сайт.ua/contacts/)
- [FAQ](https://ваш-сайт.ua/faq/)
```

Специфікація: [llmstxt.org](https://llmstxt.org)

## robots.txt для AI-ботів

У 2026 році важливо явно дозволити або заблокувати AI-crawlers у `robots.txt`:

```txt
# Дозволити Google AI
User-agent: Google-Extended
Allow: /

# Дозволити ChatGPT
User-agent: GPTBot
Allow: /

# Дозволити Claude
User-agent: ClaudeBot
Allow: /

# Дозволити Perplexity
User-agent: PerplexityBot
Allow: /
```

Якщо не хочете щоб AI-системи використовували ваш контент для навчання — можна заблокувати `Google-Extended` та `GPTBot`, але це може зменшити AI-visibility.

## Важливо

AI-системи та Google повинні бачити основний контент без проблем.

Не можна, щоб важливий текст:

- завантажувався тільки після кліку;
- був прихований у JS;
- з’являвся лише після hydration;
- був у зображеннях без текстової версії;
- був недоступний для crawler.

# 13. FAQ Strategy

FAQ — один з найважливіших блоків для AI Search.

## Для кожної сторінки потрібно 5–10 питань

Типи питань:

- скільки коштує;
- скільки часу займає;
- які етапи;
- які ризики;
- чи потрібні матеріали;
- чи є гарантія;
- які помилки бувають;
- що входить у послугу;
- коли потрібен спеціаліст.

## Приклад

### Скільки коштує підключення генератора?

Вартість підключення генератора залежить від потужності, типу підключення, наявності АВР, стану електрощита та складності робіт. Точну ціну можна визначити після огляду об’єкта.

# 27. Фінальний чеклист AI Search

AI має за 15 секунд зрозуміти:

- хто ви;
- що продаєте;
- де працюєте;
- які послуги надаєте;
- скільки це коштує;
- чому вам можна довіряти;
- як з вами зв’язатися;
- які сторінки пов’язані між собою;
- чому ваш сайт експертний у темі.

Якщо AI цього не розуміє — сайт треба доопрацьовувати.

# 28. Головне правило 2026

У 2026 році виграють не сайти з найбільшою кількістю тексту.

Виграють сайти, які:

- дають точні відповіді;
- мають сильну структуру;
- мають експертність;
- мають довіру;
- мають topical authority;
- мають правильну технічну базу;
- легко читаються AI-системами.

# 30. Короткий висновок

AI Search Optimization — це не окрема магічна техніка. Це правильне поєднання класичного SEO, технічної якості, експертного контенту, структури, довіри та зручності для AI-систем.
