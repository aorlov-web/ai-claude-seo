## 1. КОНТЕКСТ 2026: ЩО ЗМІНИЛОСЬ

### 1.1 Нова реальність пошуку

```
РАНІШЕ (до 2024):            ЗАРАЗ (2026):
─────────────────────         ─────────────────────
Ранжування у SERP        →   Цитування в AI відповідях
Кількість ключових слів  →   Глибина покриття теми
Backlinks = авторитет    →   E-E-A-T + topical authority
CTR з синіх лінків       →   AI Citation + brand mentions
Окремі сторінки          →   Content clusters
```

### 1.2 Критичні цифри 2026

| Факт | Цифра | Джерело |
|------|-------|---------|
| Пошуків в Google AI Mode без кліку | **93%** | Semrush, вересень 2025 |
| AI Overviews видно у всіх запитах | **4.5–12.5%** | Google |
| CTR зростає якщо сайт цитується в AI Overview | **+35%** | Дослідження 2025 |
| "How to do X" запити охоплені AI | **99.9%** | |
| "Buy X / Best X / Price X" охоплені AI | **3–4%** | |
| Трафік affiliate сайтів після грудень 2025 core update | **-71%** | |

### 1.3 Грудень 2025 Core Update — що означає

- **E-E-A-T тепер застосовується до ВСІХ конкурентних запитів** (не тільки YMYL)
- Анонімний або генеричний авторський контент — пенальті навіть для non-YMYL тем
- AI-визначення контенту значно покращилось — Google відрізняє ChatGPT-текст
- Нові spam категорії: expired domain abuse, site reputation abuse, scaled content abuse

### 1.4 Дві паралельні стратегії (обидві обов'язкові)

```
SEO (традиційний)                    GEO (Generative Engine Optimization)
────────────────                     ────────────────────────────────────
Ранжування в blue links         +    Цитування в AI Overviews, AI Mode
On-page optimization            +    Answer-first formatting
Backlinks                       +    Entity signals, brand mentions
Keywords                        +    Quotable facts, statistics
Internal linking                +    Clear Q&A structure
Mobile + CWV                    +    Schema + structured data
```

## 2. ПРОЦЕС РОБОТИ З КОНТЕНТОМ

### 2.1 Старт будь-якої контент-задачі

```
КРОК 1 — Визначити тип сторінки
  □ Homepage / Service / Location / Blog / About / Landing?
  → Визначає мінімум слів, структуру, schema тип

КРОК 2 — Дослідити пошуковий намір (search intent)
  □ Informational: "що таке SEO"
  □ Commercial: "найкраща SEO стратегія"
  □ Transactional: "замовити SEO послуги"
  □ Navigational: "Ваш бренд контакти"
  → Контент повинен ТОЧНО відповідати наміру

КРОК 3 — Перевірити поточний стан
  □ Скільки слів зараз?
  □ Які H1/H2/H3 заголовки?
  □ Чи є schema markup?
  □ E-E-A-T сигнали присутні?

КРОК 4 — Знайти топ-3 конкурентів
  □ Що вони мають що ми ні?
  □ Які секції вони використовують?
  □ Яка глибина їхнього контенту?

КРОК 5 — Написати / оновити контент
  □ Слідувати структурі (Секція 5)
  □ Слідувати E-E-A-T стандартам (Секція 3)
  □ Додати GEO елементи (Секція 4)

КРОК 6 — Перевірити за чеклістом (Секція 11)

КРОК 7 — Оновити schema, дати, internal links
```

### 2.2 Мінімальний обсяг за типом сторінки

| Тип сторінки | Мінімум слів | Унікальність | Оновлення |
|-------------|-------------|-------------|-----------|
| Homepage | 500 | 100% | Щоквартально |
| Service page | 800 | 100% | Щоквартально |
| Location (primary) | 600 | 60%+ | Щоквартально |
| Location (secondary) | 500 | 40%+ | Раз на рік |
| Blog post | 1 500 | 100% | Раз на рік |
| About page | 600 | 100% | При змінах |
| Landing page | 600 | 100% | При змінах |
| FAQ page | 800 | 100% | Щоквартально |

> **Важливо:** Це мінімум для покриття теми, не ціль. Google не ранжує за кількістю слів.
> 600-слівна сторінка що точно відповідає на запит > 2000-слівна що розлита водою.

### 2.3 Заборонено без обговорення

- ❌ Масово генерувати контент AI без людської редакції та E-E-A-T сигналів
- ❌ Копіювати структуру/текст між сторінками (навіть між містами)
- ❌ Публікувати без schema markup
- ❌ Публікувати без дати оновлення
- ❌ Змінювати URL після індексації без 301 redirect
- ❌ noindex на сторінках що мають трафік без перевірки
- ❌ Видаляти контент що має backlinks

## 3. E-E-A-T СТАНДАРТИ

### 3.1 Ваги факторів (Google QRG Sept 2025)

| Фактор | Вага | Що означає |
|--------|------|-----------|
| **Trustworthiness** | 30% | Найважливіший. Прозорість, контакти, реальна компанія |
| **Expertise** | 25% | Кваліфікація, глибина знань, точність |
| **Authoritativeness** | 25% | Визнання іншими, цитування, awards |
| **Experience** | 20% | Перший-особовий досвід — ГОЛОВНИЙ диференціатор від AI |

### 3.2 Experience — чому це головне в 2026

AI може симулювати Expertise. AI НЕ може симулювати справжній досвід.

**Сигнали Experience (додавати обов'язково):**
```
✅ "We tested X with 50+ clients and found..."
✅ Оригінальні скриншоти, фото процесу
✅ Конкретні кейси: клієнт + проблема + дія + результат + дата
✅ "In our experience working with [Місто] restaurants..."
✅ Before/after дані з реальних проєктів
✅ Помилки що ми зробили і що вивчили (дуже довіряємо)
✅ Специфічні деталі що неможливо вигадати
```

### 3.3 Expertise сигнали

```
✅ Author bio з credentials на кожній сторінці/статті
✅ Certifications: профільні сертифікації та партнерства, etc.
✅ Посилання на авторитетні джерела (Google, Stats [Країна], industry reports)
✅ Технічна точність — терміни використовуються правильно
✅ Дати оновлення контенту
✅ Посилання між статтями (топічна глибина)
```

### 3.4 Authoritativeness сигнали

```
✅ Clutch.co, G2, Trustpilot профіль і рейтинг
✅ Згадки в медіа (навіть місцевих)
✅ Case studies з реальними назвами клієнтів (або з дозволу)
✅ Awards і certifications видимі на сторінці
✅ Кількість клієнтів, проєктів, років роботи
✅ Speakers, publications, interviews
```

### 3.5 Trustworthiness сигнали

```
✅ Фізична адреса (обов'язково для local business)
✅ Телефон у E.164 форматі: +10000000000
✅ Privacy Policy, Terms of Service, Cookie Policy
✅ HTTPS (обов'язково)
✅ Реальні відгуки з іменами і датами
✅ Transparent pricing (або чітке "get a quote")
✅ Прозора команда з фото і LinkedIn
```

### 3.6 AI Content — правила

```
ПРИЙНЯТНО:
✅ AI допомагає структурувати і писати чернетку
✅ Людина додає E-E-A-T сигнали, перевіряє факти
✅ Унікальні інсайти, дані, досвід додаються вручну
✅ Author attribution присутній

ЗАБОРОНЕНО (пенальті):
❌ Публікувати AI текст без редакції
❌ Генерувати 100+ сторінок з однаковою структурою
❌ AI контент без author attribution
❌ Фактичні помилки без перевірки
```

## 4. GEO — ОПТИМІЗАЦІЯ ДЛЯ AI

### 4.1 Цільові AI платформи

| Платформа | Як цитує | Що оптимізувати |
|-----------|---------|----------------|
| **Google AI Overviews** | З індексованих сторінок | Schema, структура, E-E-A-T |
| **Google AI Mode** | Conversational synthesis | Чіткі факти, Q&A, entities |
| **ChatGPT / GPT Search** | Training + web search | Клір statements, brand mentions |
| **Perplexity** | Real-time web crawl | Citations, structured data, headings |
| **Bing Copilot** | Bing index + GPT-4 | Bing indexation, structured data |
| **Apple Intelligence** | Web + Safari crawl | Applebot доступ, structured content |

### 4.2 Answer-First Formatting

Найважливіший GEO принцип: **відповідай на питання в перших 2-3 реченнях.**

```markdown
ПОГАНО:
"SEO is a complex discipline that has been evolving since the early days of the internet.
There are many factors that influence how websites rank in search engines..."

ДОБРЕ:
"SEO (Search Engine Optimization) is the process of optimizing your website to rank
higher in Google search results. For [Місто] businesses, effective SEO typically
generates 3-5x more qualified leads than paid ads at 1/4 the long-term cost."
```

### 4.3 Quotable Facts — формат для AI цитування

AI системи люблять цитувати конкретні, верифіковані твердження.

```
ФОРМУЛА: [Конкретний факт] + [Число/відсоток] + [Контекст]

Приклади:
✅ "[Місто] businesses that invest in local SEO see an average 147% increase
   in website traffic within 6 months (Ваш бренд client data, 2025)."

✅ "78% of local mobile searches result in a purchase within 24 hours
   (Google, 2025)."

✅ "The average cost per lead from SEO in [Місто] is $23, compared to $89
   from Google Ads — a 74% saving (Ваш бренд benchmark, 2025-2026)."
```

### 4.4 Q&A Структура для AI

Кожна сторінка має відповідати на питання у форматі питання-відповідь:

```html
<!-- Структура що добре парситься AI -->
<h2>How much does SEO cost in [Місто]?</h2>
<p>SEO services in [Місто] typically cost between $1,500–$5,000/month for small-to-medium
businesses. Ваш бренд Business offers packages starting at $1,497/month, with ROI
typically visible within 3–6 months through increased organic traffic and lead generation.</p>
```

### 4.5 Entity Signals — важливо для ChatGPT

```
Entities що мають бути чітко визначені на сайті:

Organization:
  name: "Ваш бренд Business" (завжди повна назва)
  url: example.com
  foundingDate: 2020
  address: [Місто], [Країна]
  telephone: +10000000000

Services:
  Кожна послуга — окрема entity з чітким визначенням

People:
  Founder/CEO з повним ім'ям, LinkedIn, роль

Locations:
  Кожне місто обслуговування з координатами
```

### 4.6 llms.txt — AI instruction file

Файл `/llms.txt` в корені проєкту:
```
# Ваш бренд Business
> AI-powered digital business in [Місто], [Країна].
Founded in 2020. Serving 500+ бізнеси.

## Key Pages
- [Services](/services): SEO, Google Ads, AI Automations, Web Design
- [About](/about): Our team, story, and approach
- [Locations](/locations): 26+ cities across [Країна]
...
```

## 5. СТРУКТУРА КОНТЕНТУ

### 5.1 Шаблон Service Page (800–1 400 слів)

```
H1: [Primary Keyword] in [City] — [Differentiator]
   Приклад: "SEO Services in [Місто] — AI-Powered, Results-Guaranteed"

[Intro paragraph — 80-100 слів]
   - Primary keyword у першому реченні
   - Проблема яку вирішує послуга
   - Чому Ваш бренд — правильний вибір
   - 1 конкретна цифра або результат

H2: What's Included in Our [Service] Package
   H3: [Feature 1] — 50-80 слів + що клієнт отримає
   H3: [Feature 2] — 50-80 слів
   H3: [Feature 3] — 50-80 слів
   (4-6 features total)

H2: [Service] Results for [City] Businesses
   - 2-3 анонімних або реальних кейси
   - Формат: Ніша → Проблема → Дія → Результат + дата

H2: Our [Service] Process
   - Крок 1, 2, 3... з часовими рамками
   - Це і E-E-A-T і GEO signal

H2: [Service] Pricing in [City]
   - Або конкретні ціни, або "Starting from $X"
   - Пояснення що входить

H2: Frequently Asked Questions
   - 6-8 питань (відповіді 50-100 слів кожна)
   - Питання у форматі як люди реально питають

H2: Ready to [Outcome]? — CTA секція
   - Заклик до дії
   - Контактна форма або кнопка

Footer internal links:
   - Nearby locations (для local pages)
   - Related services
```

```
   - Hook: surprising stat або bold statement

H2: [Section 1 — Context/Problem]
H2: [Section 2 — Main topic]

H2: [Conclusion + CTA]

Author block:
   - Ім'я, посада, фото
   - Published: [date] | Updated: [date]
```

## 6. [Регіон]-PAGE SEO СТАНДАРТИ

### 6.1 Title Tag

```
Формула: [Primary Keyword] | [Secondary Context] — [Brand]
або:      [Primary Keyword] in [City] | [Brand Name]

Довжина: 50–60 символів (Google обрізає після ~60)

Приклади:
✅ "SEO Services [Місто] | Ваш бренд"           (38 символів)
✅ "AI Automation Business in [Місто] | Ваш бренд"       (44 символи)
❌ "Best SEO Services Services [Місто] [Країна] Digital Marketing" (занадто довго)
❌ "Home" (занадто коротко)

Правила:
- Primary keyword ближче до початку
- Назва бренду в кінці
- Без | або — в середині якщо не потрібно
- SEO, AI — завжди великими
```

### 6.2 Meta Description

```
Довжина: 120–155 символів
Формула: [Primary keyword] + [Unique value prop] + [CTA]

Приклади:
✅ "Expert SEO Services in [Місто], [Регіон]. Ваш бренд Business delivers
   measurable results — higher rankings, more leads, better ROI.
   Free strategy session."  (155 символів)

Правила:
- Primary keyword природно вписаний
- Конкретна вигода (не "world-class services")
- CTA: "Free audit", "Get started", "Book a call"
- Унікальний для кожної сторінки
```

### 6.3 Заголовки (H1–H4)

```
H1: Один на сторінці. Primary keyword. 40-70 символів.
H2: Основні секції. Secondary keywords. Описові.
H3: Підсекції. Long-tail keywords. Конкретні.
H4: Деталі. Тільки якщо потрібні.

Правила:
- Не пропускати рівні (H1 → H3 без H2 — помилка)
- Кожен H2 має бути змістовним питанням або твердженням
- Ключові слова природно, не forcefully
- SEO, AI, GEO — завжди правильний регістр
```

### 6.4 Keyword Density

```
Primary keyword: 1–2% (приблизно 1 раз на 100 слів)
LSI / semantic keywords: природно в тексті
Заборонено: keyword stuffing, repeat 3+ рази в абзаці

Semantic keywords (для SEO Services):
- "search engine optimization"
- "organic traffic"
- "local search"
- "Google rankings"
- "digital marketing"
- "[city] businesses"
```

### 6.5 Internal Linking

| Тип сторінки | Кількість internal links |
|-------------|------------------------|
| Blog post (1500+ слів) | 5–8 лінків |
| Service page | 3–5 лінків |
| Location page | 4–6 лінків |
| Homepage | 8–12 лінків |

```
Правила:
✅ Descriptive anchor text: "our SEO services in [Місто]"
❌ Generic anchor: "click here", "read more", "this page"
✅ Лінки на релевантний контент (не просто щоб мати лінки)
✅ Лінки з нових сторінок на встановлені (передача авторитету)
✅ Немає orphan pages (кожна сторінка має хоча б 1 inbound link)
```

## 7. TOPICAL AUTHORITY

### 7.1 Content Cluster Architecture

```
PILLAR PAGE (2000–3000 слів)
│   Головна тема: "SEO Services [Місто]"
│   Широке покриття всієї теми
│
├── CLUSTER PAGE 1: "Technical SEO [Місто]"
├── CLUSTER PAGE 2: "Local SEO [Місто]"
├── CLUSTER PAGE 3: "SEO Pricing [Місто]"
├── CLUSTER PAGE 4: "SEO vs Google Ads [Місто]"
├── CLUSTER PAGE 5: "How Long Does SEO Take"
├── CLUSTER PAGE 6: "[Місто] SEO Case Studies"
├── CLUSTER PAGE 7: "[Місто] SEO Statistics 2026"
└── CLUSTER PAGE 8: "DIY vs Business SEO [Місто]"

Кожна cluster page:
  - Лінкується на pillar page
  - Pillar page лінкується назад
  - Cluster pages лінкуються між собою де релевантно
```

### 7.2 Правило 1 Cluster = 1 Topic

```
НЕ намагатись покрити все однією сторінкою.

Замість: "SEO Guide for Everything" (1 сторінка)
Краще:   "Technical SEO Guide" + "Content SEO Guide" +
         "Link Building Guide" + "Local SEO Guide"
         (4 cluster pages + 1 pillar)
```

## 8. SCHEMA MARKUP

### 8.1 Schema по типу сторінки

| Тип сторінки | Primary Schema | Додатково |
|-------------|---------------|-----------|
| Homepage | `Organization` + `WebSite` + `SiteLinksSearchBox` | `LocalBusiness` якщо є |
| Service page | `Service` або `LocalBusiness` | `FAQPage` |
| Location page | `LocalBusiness` + `GeoCoordinates` | `Service`, `FAQPage` |
| Blog post | `BlogPosting` або `Article` | `BreadcrumbList` |
| About page | `Organization` + `Person` (team) | |
| Contact page | `LocalBusiness` | `ContactPage` |

### 8.2 Organization Schema (обов'язковий на homepage)

```json
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "Ваш бренд Business",
  "url": "https://example.com",
  "foundingDate": "2020",
  "description": "AI-powered digital business in [Місто], [Країна].",
  "telephone": "+10000000000",
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "240 Richmond St W",
    "addressLocality": "[Місто]",
    "addressRegion": "[Регіон]",
    "addressCountry": "CA"
  },
  "areaServed": ["[Місто]", "[Місто]", "[Місто]", "[Країна]"],
  "sameAs": [
    "https://www.linkedin.com/company/boomy-marketing",
    "https://example.com/google-profile"
  ],
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.9",
    "reviewCount": "47",

  }
}
```

> **Важливо:** aggregateRating тільки якщо відгуки видимі на сторінці!

```json
{
```

```json

  "author": {
```

### 8.5 FAQPage Schema

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "How much does SEO cost in [Місто]?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "SEO services in [Місто] typically cost $1,500–$5,000/month..."
      }
    }
  ]
}
```

> **Примітка:** FAQ schema для Google rich results тільки для government і healthcare (серпень 2023).
> Для комерційних сайтів — зберігаємо для AI/LLM citation benefit і інших платформ.

## 9. LOCAL SEO КОНТЕНТ

### 9.1 Обов'язкові елементи кожної location page

```
□ Назва міста в H1
□ Провінція в тексті ([Регіон], [Регіон], [Регіон], etc.)
□ Назва міста 5–8 разів природно в тексті
□ GeoCoordinates в schema
□ areaServed в schema
□ Список сусідніх міст з лінками (6 міст)
□ Місто-специфічний FAQ (мінімум 2 з 6 питань)
□ NAP: Name, Address, Phone — ідентично скрізь
```

### 9.2 Локальна специфіка (що робить сторінку унікальною)

```
Замість генеричного тексту — місто-специфічний:

[Місто]:
  - Найбільший ринок Канади, 200K+ бізнесів
  - Топ галузі: real estate, finance, restaurants, tech
  - Neighbourhoods: Downtown, Midtown, North York, Scarborough

[Місто]:
  - [Регіон] tech hub, strong tourism industry
  - Bilingual considerations (English/French less so, but Mandarin/Cantonese)
  - Топ галузі: tech, tourism, real estate, restaurants

[Місто]:
  - Alberta oil & gas, growing tech sector
  - Топ галузі: energy, agriculture, real estate, retail

[Додавати для кожного нового міста]
```

### 9.3 NAP Consistency — залізне правило

```
СКРІЗЬ однаково:
  Name:      "Ваш бренд Business"
  Phone:     +10000000000
  Address:   240 Richmond St W, [Місто], [Регіон], CA

Перевіряти в:
  □ Кожній HTML сторінці
  □ Schema markup
  □ Google Business Profile
  □ llms.txt
  □ Всі external citations/directories
```

### 9.4 Location Page — порогові значення

| Кількість сторінок | Статус | Вимога |
|-------------------|--------|--------|
| < 30 | ✅ Безпечно | Базова унікальність |
| 30–50 | ⚠️ Увага | 60%+ унікального контенту |
| 50+ | 🛑 Ризик | Потрібне обґрунтування + локальні сигнали |

## 10. ТЕХНІЧНІ СИГНАЛИ

### 10.1 Core Web Vitals цілі (2026)

| Метрика | Ціль | Тривожно | Критично |
|---------|------|---------|---------|
| LCP | ≤ 2.5s | 2.5–4s | > 4s |
| INP | ≤ 200ms | 200–500ms | > 500ms |
| CLS | ≤ 0.1 | 0.1–0.25 | > 0.25 |
| TTFB | < 800ms | 800ms–1.8s | > 1.8s |

> INP замінив FID з березня 2024. FID більше не існує.
> Google використовує 75-й перцентиль реальних користувачів (CrUX), не Lighthouse.

### 10.2 Індексація — перевіряти

```
□ robots.txt — не блокує важливі сторінки
□ AI боти дозволені: GPTBot, ClaudeBot, Google-Extended,
  PerplexityBot, OAI-SearchBot, Applebot-Extended
□ sitemap.xml — актуальний (lastmod = дата останньої зміни)
□ Canonical URLs — без trailing slash конфліктів
□ noindex — тільки там де потрібно (search, thank-you pages)
□ Зламані посилання — 0 (404 = crawl budget waste)
```

### 10.3 Mobile-First (100% з липня 2024)

```
Google індексує ТІЛЬКИ мобільну версію.

Перевіряти:
□ Весь контент видимий на mobile (не hidden)
□ Schema markup однакова на mobile і desktop
□ Meta tags однакові на mobile і desktop
□ Зображення оптимізовані для mobile
□ Font size мінімум 16px для основного тексту
□ Tap targets мінімум 44x44px
```

## 11. ЧЕКЛІСТИ

### 11.1 Перед публікацією нової сторінки

```
[Регіон]-PAGE:
□ H1 містить primary keyword
□ Title tag 50-60 символів, keyword на початку
□ Meta description 120-155 символів з CTA
□ Primary keyword в першому реченні
□ Слова: мінімум за типом сторінки (Секція 2.2)
□ Заголовки H1→H2→H3 без пропусків
□ Internal links: 3-5 для service/location, 5-8 для blog
□ Дата публікації або оновлення видима

E-E-A-T:
□ Мінімум 1 статистика з джерелом
□ Мінімум 1 конкретний результат з числами
□ Experience signal (перша особа або кейс)
□ Author attribution або organization credit

GEO:
□ Answer-first параграф в intro
□ Мінімум 2 quotable facts (число + контекст)
□ FAQ секція з 5-6 питань у форматі як питають люди
□ Entity "Ваш бренд Business" чітко названа

TECHNICAL:
□ Schema markup відповідного типу додано і валідно
□ Зображення мають alt text (10-125 символів)
□ Всі зображення WebP
□ Canonical URL правильний (без .html, без trailing slash якщо vercel.json каже)
□ Сторінка додана в sitemap.xml з правильним lastmod
```

### 11.2 При оновленні існуючої сторінки

```
□ Оновити dateModified в schema
□ Оновити дату "Last Updated" в HTML якщо видима
□ Оновити sitemap.xml lastmod
□ Перевірити чи всі internal links ще дійсні
□ Перевірити чи факти/числа актуальні
□ Запросити повторну індексацію в GSC (якщо критична сторінка)
```

### 11.3 Щомісячний контент аудит


## ДОВІДКА — Типові помилки і виправлення

| Помилка | Наслідок | Виправлення |
|---------|---------|------------|
| Keyword stuffing | Пенальті | Max 1-2% density, semantic variations |
| Тонкий контент | Не ранжується | Мінімум за Секцією 2.2 |
| Немає E-E-A-T сигналів | Грудень 2025 пенальті | Додати experience signals, author |
| Однаковий контент між містами | Doorway page пенальті | 40-60% унікальний контент |
| Застарілі дати | Freshness пенальті | Оновлювати dateModified |
| Немає schema | Missed rich results + AI | Додати відповідний тип |
| Generic AI контент | Низький E-E-A-T | Людська редакція + experience signals |
| FAQ без видимих відповідей | Не буде AI citation | Видимі Q&A на сторінці |
| FID замість INP | Неправильні метрики | INP з березня 2024 |
| aggregateRating без відгуків | Google Manual Action | Видалити або додати реальні відгуки |
